   - Dependencies: chromadb, langchain-text-splitters, openai (existing)

2. **## Directory Structure**
   ```
   rag/
   ├── __init__.py           # Public API exports
   ├── embeddings.py         # DashScope embedding client
   ├── vector_store.py       # ChromaDB wrapper
   ├── document_processor.py # Chunking and indexing
   ├── retriever.py          # Similarity search
   └── loaders/              # Data source connectors

       ├── file_loader.py    # PDF/TXT/MD/DOCX

       └── api_loader.py     # REST API fetcher
   ```

3. **## Core Components**
   For each component, specify:
   - Class name and responsibility
   - Key methods with signatures
   - Dependencies (imports)
   - Threading model (sync/async)

   Components:
   - `DashScopeEmbedding` (embeddings.py)
   - `VectorStore` (vector_store.py)
   - `DocumentProcessor` (document_processor.py)
   - `RAGRetriever` (retriever.py)
   - `FileLoader`, `DBLoader`, `APILoader` (loaders/)

4. **## Integration Points**
   - How to extend registry.py with "rag" category
   - How llm.py will call RAG components
   - Configuration flow from config.py

5. **## Data Flow**
   ```
   Document → Loader → Processor → Embedding → VectorStore
                                                     ↓
   Query → Embedding → VectorStore → Retriever → LLM Context
   ```

   - Document ingestion: Background thread (CPU-bound + API calls)
   - Query retrieval: Sync call in LLM thread (must not block streaming)
   - Target: Retrieval under 300ms to maintain streaming UX

   - Logging: Use utils.logger pattern
   - Timeout: 300ms for retrieval, skip if exceeded
