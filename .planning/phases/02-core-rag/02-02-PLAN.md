# Phase 2 Plan 02: ChromaDB 向量存储

**目标:** 实现 ChromaDB 向量存储封装，提供文档持久化和相似度检索。

## 背景

根据 Phase 1 验证结果:
- ChromaDB 版本: 1.5.9
- Query P50 延迟: 0.59 ms
- Query P95 延迟: 2.2 ms
- 距离度量: cosine

## 任务

### Task 1: 实现 VectorStore 类

<read_first>
- rag/__init__.py - 查看现有 VectorStoreProtocol 定义
- rag/API_DESIGN.md - 查看完整 API 设计
- .planning/phases/01-research-design/01-VERIFICATION.md - 查看验证结果
</read_first>

<action>
创建 `rag/vector_store.py` 文件，实现 VectorStore 类:

1. 实现 `__init__` 方法:
   - 参数: persist_dir="./data/chromadb", collection_name="knowledge_base", distance_metric="cosine"
   - 创建 chromadb.PersistentClient
   - 获取或创建 collection（使用 get_or_create_collection）

2. 实现 `add(chunks, embeddings, metadatas=None, ids=None) -> list[str]` 方法:
   - 如果 ids 为 None，自动生成 ID（使用 f"doc_{i}_{hash}" 格式）
   - 调用 collection.add() 添加文档
   - 返回添加的文档 ID 列表

3. 实现 `query(query_embedding, top_k=3, filter_metadata=None) -> list[dict]` 方法:
   - 调用 collection.query() 查询相似文档
   - 返回格式: [{"text": str, "metadata": dict, "distance": float}, ...]

4. 实现 `delete(ids) -> None` 方法:
   - 调用 collection.delete() 删除文档

5. 实现 `count() -> int` 方法:
   - 返回 collection.count()

6. 错误处理:
   - 捕获 chromadb 异常并转换为 RuntimeError
</action>
