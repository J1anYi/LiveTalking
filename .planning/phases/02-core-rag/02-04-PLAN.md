# Phase 2 Plan 04: RAG 检索器

**目标:** 实现端到端 RAG 检索服务，协调 Embedding 和 VectorStore。

## 背景

RAGRetriever 是核心组件，负责:
1. 接收用户查询
2. 生成查询向量
3. 检索相似文档
4. 支持文档摄入

## 任务

### Task 1: 实现 RAGRetriever 类

<read_first>
- rag/__init__.py - 查看现有定义
- rag/embeddings.py - Embedding 客户端接口
- rag/vector_store.py - VectorStore 接口
- rag/API_DESIGN.md - 查看完整 API 设计
</read_first>

<action>
创建 `rag/retriever.py` 文件:

1. 实现 `__init__` 方法:
   - 参数: vector_store, embedding_client, top_k=3, timeout_ms=300, min_similarity=0.0
   - 存储所有配置

2. 实现 `retrieve(query, top_k=None, filter_metadata=None) -> list[dict] | None` 方法:
   - 使用 embedding_client.embed_query() 生成查询向量
   - 使用 vector_store.query() 检索文档
   - 如果 min_similarity > 0，过滤低相似度结果
   - 超时处理：使用 timeout_ms 限制
   - 错误处理：返回 None 而不是抛出异常

3. 实现 `ingest(documents, batch_size=20) -> int` 方法:
   - documents 格式: [{"text": str, "metadata": dict}, ...]
   - 批量处理：每 batch_size 个文档一批
   - 对每批文档:
     - 提取文本
     - 调用 embedding_client.embed() 生成向量
     - 调用 vector_store.add() 存储
   - 返回添加的文档总数

4. 实现 `clear() -> None` 方法:
   - 清空向量存储中的所有文档
</action>

<acceptance_criteria>
- `rag/retriever.py` 文件存在
- 文件包含 `class RAGRetriever:`
- 文件包含 `def retrieve(self, query, top_k=None, filter_metadata=None)`
- 文件包含 `def ingest(self, documents, batch_size=20)`
- 文件包含 `def clear(self)`
- 返回类型注解包含 `list[dict] | None`
</acceptance_criteria>
