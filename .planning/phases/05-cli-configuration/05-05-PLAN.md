# RAG 知识库使用指南

## 概述

RAG (Retrieval-Augmented Generation) 知识库功能为 LiveTalking 数字人提供知识检索能力，让数字人能够基于知识库内容回答问题。

## 配置方式

RAG 支持三种配置方式，优先级从高到低：

1. **CLI 参数** - 启动时指定
2. **环境变量** - 容器化部署场景
3. **配置文件** - YAML 格式

### 配置优先级

```
CLI 参数 > 环境变量 > 配置文件
```

例如：
- 配置文件设置 `top_k: 3`
- 环境变量设置 `RAG_TOP_K=5`
- CLI 设置 `--rag_top_k 10`
- 最终使用 `top_k=10`（CLI 优先级最高）

## CLI 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--rag_enabled` | flag | False | 启用 RAG 知识库 |
| `--rag_top_k` | int | 3 | 检索文档数量 |
| `--rag_persist_dir` | str | ./data/chromadb | 向量库目录 |
| `--rag_collection` | str | knowledge_base | 集合名称 |
| `--rag_config` | str | data/rag_config.yaml | 配置文件路径 |

### 使用示例

```bash
# 启用 RAG，使用默认配置
python app.py --rag_enabled

# 指定检索数量和向量库路径
python app.py --rag_enabled --rag_top_k 5 --rag_persist_dir /data/my_chromadb

# 使用自定义配置文件
python app.py --rag_config /path/to/my_config.yaml
```

## 环境变量

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `RAG_ENABLED` | bool | 启用 RAG (true/false/1/0/yes/no) |
| `RAG_TOP_K` | int | 检索文档数量 |
| `RAG_PERSIST_DIR` | str | 向量库目录 |
| `RAG_COLLECTION` | str | 集合名称 |

### 使用示例

```bash
# 设置环境变量
export RAG_ENABLED=true
export RAG_TOP_K=5
python app.py

# 或在 Docker 中
docker run -e RAG_ENABLED=true -e RAG_TOP_K=5 livetalking:latest
```

## 配置文件

配置文件使用 YAML 格式，默认路径 `data/rag_config.yaml`。

### 配置文件格式

```yaml
rag:
  enabled: true
  top_k: 3
  persist_dir: ./data/chromadb
  collection: knowledge_base
```

### 使用步骤

1. 复制示例配置：
   ```bash
   cp data/rag_config.yaml.example data/rag_config.yaml
   ```

2. 编辑配置文件：
   ```bash
   vim data/rag_config.yaml
   ```

3. 启动服务：
   ```bash
   python app.py
   ```

## 知识库管理

### 导入文档

```python
from rag import FileLoader, DocumentProcessor, VectorStore

# 加载文档
loader = FileLoader(path="./data/knowledge")
documents = loader.load()

# 处理并入库
processor = DocumentProcessor()
store = VectorStore(persist_dir="./data/chromadb")

for text, metadata in documents:
    chunks = processor.chunk(text)
    embeddings = processor.embed(chunks)
    store.add(chunks, embeddings, [metadata] * len(chunks))
```

### 查看知识库状态

```python
from rag import VectorStore

store = VectorStore(persist_dir="./data/chromadb")
print(f"文档数量: {store.count()}")
```

## 常见问题

### Q: RAG 检索延迟太高？

A: 检查向量库大小和网络连接（使用 DashScope Embedding API）。可以考虑：
- 减小 `top_k` 值
- 使用本地 Embedding 模型（需要额外配置）

### Q: 如何切换不同的知识库？

A: 使用不同的 `collection` 或 `persist_dir`：
```bash
python app.py --rag_collection product_kb --rag_persist_dir ./data/product_kb
```

### Q: 配置不生效？

A: 检查优先级：
1. CLI 参数优先级最高，会覆盖其他配置
2. 环境变量优先级中等
3. 配置文件优先级最低
