    ## 概述

    RAG (Retrieval Augmented Generation) 知识库模块为 LiveTalking 数字人提供智能知识检索功能。

    ## 快速开始

    ### 环境要求
    - Python 3.10+
    - DASHSCOPE_API_KEY 环境变量

    ### 基本用法
    1. 启用 RAG 功能
    ```bash
    python app.py --rag_enabled --rag_top_k 3
    ```

    2. 使用配置文件
    ```bash
    cp data/rag_config.yaml.example data/rag_config.yaml
    # 编辑配置文件
    python app.py --rag_config data/rag_config.yaml
    ```

    ### 配置优先级
    CLI 参数 > 环境变量 > 配置文件 > 默认值

    ## 安装依赖

    pip install chromadb dashscope pyyaml python-docx

    ## 配置说明

    ### CLI 参数
    | 参数 | 说明 | 默认值 |
    |------|------|--------|
    | --rag_enabled | 启用知识库检索 | False |
    | --rag_top_k | 检索文档数量 | 3 |
    | --rag_persist_dir | 向量存储路径 | ./data/chromadb |

    ### 环境变量
    | 变量 | 说明 |
    |------|------|
    | RAG_ENABLED | 启用知识库 (true/false) |
    | RAG_TOP_K | 检索数量 |
    | RAG_PERSIST_DIR | 存储路径 |
    | RAG_COLLECTION | 集合名称 |

    ## API 参考

    ### 核心类

    #### RAGRetriever
    检索器，管理知识库的索引和检索。
    ```python
    from rag import RAGRetriever, VectorStore, DashScopeEmbedding

    store = VectorStore(persist_dir="./data/chromadb")
    embedding = DashScopeEmbedding(api_key="your-api-key")
    retriever = RAGRetriever(store, embedding, top_k=3)

    # 检索
    results = retriever.retrieve("什么是 RAG？")

    # 索引文档
    retriever.ingest([{"text": "文档内容", "metadata": {}}])
    ```

    #### VectorStore
    向量存储，基于 ChromaDB。
    ```python
    from rag import VectorStore

    store = VectorStore(persist_dir="./data/chromadb")
    store.add(chunks=["文本"], embeddings=[[0.1, ...]])
    results = store.query(query_embedding=[0.1, ...], top_k=3)
    ```

    #### DashScopeEmbedding
    Embedding 生成器。
    ```python
    from rag import DashScopeEmbedding

    embedding = DashScopeEmbedding(api_key="your-api-key")
    vectors = embedding.embed(["文本1", "文本2"])
    query_vector = embedding.embed_query("查询文本")
    ```

    #### DocumentProcessor
    文档处理器，负责分块。
    ```python
    from rag import DocumentProcessor

    processor = DocumentProcessor(chunk_size=800, chunk_overlap=100)
    chunks = processor.process_text("长文本...", metadata={"source": "file.txt"})
    ```

    #### FileLoader

    from rag import FileLoader

    loader = FileLoader()
    content, metadata = loader.load("document.pdf")

    from rag import SQLiteConnector

    ## FAQ

    ### Q: 如何添加新的知识文档？
    A: 将文档放入 `data/knowledge_base/` 目录，重启应用即可自动加载。

    ### Q: 检索结果不准确怎么办？
    A: 尝试调整 `--rag_top_k` 参数增加检索数量，或优化文档分块大小。

    ### Q: 如何使用自己的 Embedding 模型？
    A: 实现 `EmbeddingClient` Protocol 接口，传递给 RAGRetriever。

    ### Q: 向量存储占用空间太大？
    A: ChromaDB 支持增量更新，可以定期清理不需要的文档。

    ### Q: 如何禁用 RAG 功能？
    A: 不传递 `--rag_enabled` 参数，或在配置文件中设置 `enabled: false`。

    ### Q: 支持哪些数据库？
    A: 目前支持 SQLite，MySQL 和 PostgreSQL 可通过扩展 BaseDatabaseConnector 实现。

    ## 故障排除

    ### 常见错误

    1. **DASHSCOPE_API_KEY 未设置**
       ```
       解决: export DASHSCOPE_API_KEY=your-api-key
       ```

    2. **ChromaDB 初始化失败**
       ```
       解决: 确保 persist_dir 目录有写入权限
       ```

    3. **文档加载失败**
       ```
       解决: 检查文件路径和编码格式
       ```
